CSS Methodology

The Trawl styleguide encourages easy to understand coding conventions for structuring large scale CSS code bases.

Coding Conventions

Before we start let's define the terminology used in this guide. 

                   
.selector { 
    property: value;
}

This code snippet is called a style definition.

Spacing

Here are a set of guidelines on how to use spaces and line-breaks to promote readable and clean code. 

                   
.my-class { 
    background: blue;
}

.my-other-class { 
    background: red;
}

An empty line between every style definition and a space between 'property' and 'value' helps to improve the readability.

Indentations (1 Tab = 4 Spaces)

Indentations are also really important to improve readability in your code. Every 'property-value' pair should have an indentation of 4 spaces. If you use a code-editor like Sublime Text you can change the tab configuration so it uses spaces instead. Seperation of 'property' and 'value' is emphasised by adding a space after the colon. The same goes for the 'selector' and the style definition body.

Example Code:

                   
.my-class { 
    padding: 10px 5px 10px 5px;
    background: blue;
    color: red;
}

Units (rem / px / %)

'rem' is used to define font-sizes. Every other size definition like 'width' or 'padding' should use 'px' or precent as unit.

The font-size of the root node is set to '16px'. That means that '1rem = 16px'.

Class naming conventions

The Trawl styleguide differentiates between three types of classes:

  • Component classes
  • State classes
  • Helper classes
Having these classes will help you to achieve a clean and organized structure inside your application.

Component classes

Component classes are used to define the custom style of your component. You use them like in the following example: 

                   
.navigation {
    width: 100%;
    height: 60px;
    background-color: #444;
    color: #fff;
}

Here we define the styling for our navigation component. This class only defines the styling for the component wrapper as you can see in the HTML snippet below. 

  
                
<nav class="navigation">
    <div class="navigation-logo"></div>
    <ul class="navigation-list list-inline">
        <li>
            <a href="/">Home</a>
        </li>
    </ul>
</nav>

To define the styling for all the elements (let's call them sub-components) inside our navigation component we prefix the needed classes with our component name.
In this case it is 'navigation'. So to define the styling for a logo inside our navigation we create a 'navigation-logo' class.

You can also mix up different component classes like in the example above where the base 'list-inline' class has been used but in combination with the 'navigation-list' sub-component class

State classes

Sometimes your component need to change it's appereance. For example you want to display a navigation item as active or an input as invalid. To do so you can use state classes. 

  
                
...
    <li class="is-active">
        <a href="/">Home</a>
    </li>
...


                
  
                
.navigation-list > li.is-active {
    border-bottom: 2px solid blue;
}

Pay attention to the naming of the class. State classes are usually prefixed with an 'is-' or a 'has-'. State classes are component or subcomponent specific. Defining global state classes is considered an antipattern which you should avoid.

Helper classes (Utility classes)

The last type of classes that we need are helper classes or also called utility classes. These styles can be used globaly inside your application and define styling for e.g. hiding or floating elements. Utility classes can (but didn't have to) use '!important' to overwrite other style-definitions.
Example: 

  
                
<nav class="navigation clearfix">
    <div class="navigation-logo pull-left">...</div>
    <ul class="list-inline navigation-list pull-right">
        ...
    </ul>
</nav>


                
  
                
.pull-left {
    float: left;
}

.clearfix::after { ... }

Here we create and use two utility classes. First the 'clearfix' helper to remove the floating inside our navigation component and second the 'pull-left' helper to float the logo to the left side.

Preprocessors

All the mentioned guidelines above can be used with vanilla CSS but the Trawl styleguide also works extremely well with Preprocessors like SCSS (recommended) or LESS.

File structure

Using SCSS we structure the styling of our application like this: 


app/
├── css/
│   ├── main.css
│   ├── main.min.css
└── scss/
    ├── _list.scss
    ├── _utility.scss
    ├── _font.scss
    ├── _navigation.scss
    ├── _component.scss
    └── main.scss // has all the other components included
