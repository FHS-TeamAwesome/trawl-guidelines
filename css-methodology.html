CSS Methodology

The Trawl styleguide encourages easy to understand coding conventions for structuring large scale CSS code bases.

Coding Conventions

Before we start let's define the terminology used in this guide. 

                   
.selector { 
    property: value;
}

This code snippet is called a style definition.

Spacing

Here are a set of guidelines on how to use spaces and line-breaks to promote readable and clean code. 

                   
.my-class { 
    background: blue;
}

.my-other-class { 
    background: red;
}

An empty line between every style definition and a space between 'property' and 'value' helps to improve the readability.

Indentations (1 Tab = 4 Spaces)

Indenations are also really important to improve readability in your code. Every 'property-value' pair should have an indentation of 4 spaces. If you use a code-editor like Sublime Text you can change the tab configuration so it uses spaces instead. Seperation of 'property' and 'value' is emphasised by adding a space after the colon. The same goes for the 'selector' and the style definition body.

Example Code:

                   
.my-class { 
    padding: 10px 5px 10px 5px;
    background: blue;
    color: red;
}

Units (rem / px / %)

'rem' is used to define font-sizes. Every other size definition like 'width' or padding should use 'px' or precent as unit.

The font-size of the root node is set to '16px'. That means that '1rem = 16px'.
